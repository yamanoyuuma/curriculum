IT　コンピュータとネットワークを利用した技術の総称

OS　アプリやデバイスを動作させるための基本となるソフトウェア
Windows マイクロソフトが開発・販売しているOS 
macOS (Mac OS X) アップルが開発・販売する、MacintoshのOS
UNIX OSの一種で、現存する中でもっとも古いOSのひとつ 安全性が高く、余計な機能が少ないので軽い
Linux OSの一種　安定的に動いて、費用がかからないというのが特徴

HTML　Webページを作るための最も基本的なマークアップ言語のひとつ　コンピュータに理解できるように文書の構造を定義することが、HTMLの最も重要な役割

CSS　ウェブページのスタイルを指定するための言語　HTMLと組合わせて使用　文書構造にデザインを施し、見栄えを整える役割　

MySQL　データベース管理システム　小さいアプリケーションから巨大なアプリケーションまですべてに対応できるほどの拡張性を持つ

PHP　動的にWebページを生成することができるサーバーサイドのスクリプト言語　データベースとの連携が容易

JavaScript　ログラミング言語であり、ユーザー側のWebブラウザと、Webサイトまたはウェブサービスの相互間のやりとりを、円滑にするために使用される

jQuery　JavaScriptでできることを、より簡単な記法で実現できように設計されたJavaScriptライブラリ

Apache Webサーバーソフトウェア 動作が軽快で安定している・使い勝手の良い機能性・環境に依存しない
・誰でも無料で使える

Bootstrap HTML／CSS／JavaScriptから構成される最も有名なWEBフレームワーク デザイン性も確保されている

SE SEの仕事には大きく分けて「要件定義」「基本設計」「詳細設計」「テスト」「保守・運用」という5つの段階がある

PG SEが作成した設計書に沿ってプログラミングを行う プログラマが担当する製造工程は、コーディングだけではなく単体試験までを含む

Webデザイナー　クライアントが思い描いているWebサイトを制作できるよう、全体の構成を考えながらデザインを構築する

CUI テキストデータのみで表現されているUI

GUI 画像やアイコンなどで表現されているUI

ファイル コンピュータにとって分かりやすい形でまとめたデータ　仕分けには「拡張子」を用いる

ディレクトリ フォルダ＝ディレクトリ　　ファイルを整理・保管する

ルート・ディレクトリ　一番上のディレクトリ

カレント・ディレクトリ　今いるディレクトリ

パス　処理の流れに対し、実際に実行されるプログラムのこと。あるいは任意のファイルやディレクトリまでの道筋

絶対パス　ルートディレクトリと呼ばれる階層構造の頂点から目的地までの経路を表す。ユーザが現在どのフォルダで作業中であっても、常に同じスタート地点から、常に同じ経路の表示になる。

相対パス　ユーザが現在作業しているフォルダから目的のフォルダまでの経路を表す

HTTP　データをサーバとクライアントの間で通信するときの通信規約　

FTP サーバーとクライアント間で、ファイルを送受信する通信の決まりごと

HTTPS 「セキュリティがしっかりしている」といった意味で使われている単語です。

SSL インターネット上におけるウェブブラウザとウェブサーバ間でのデータの通信を暗号化し、送受信させる仕組みのこと 重要な情報を、悪意ある第三者による盗聴を防いだり、送信される重要な情報の改ざんを防ぐ役割を持つ

SSH　ネットワークに接続された機器を遠隔操作し、管理するための手段

ポート、ポート番号　TCP/IP通信において、 コンピュータが通信に使用するプログラムを識別するための番号

HTTP、FTP、SSHの各ポート番号　TCP 80 : HTTP　TCP 20 : FTP (データ)　TCP 21 : FTP (制御)　TCP 22 : SSH　

ファイアウォール　企業などの内部ネットワークをインターネットを通して侵入してくる不正なアクセスから守るための“防火壁”

TCP　信頼性の高い通信を実現するために使用されるプロトコル　通信特性によりTCPまたはUDPを使い分けされる

IPアドレス　インターネット上に接続された機器が持つナンバー　ネットワーク上で通信相手を間違わないようにするために使われる

グローバルIPアドレス　インターネットに接続する機器が使用できるIPアドレス　インターネット上で通信をする場合に必要になる

プライベートIPアドレス　組織内のネットワーク（プライベートネットワーク）でのみ使用できるIPアドレス　

ドメイン　インターネットの「住所」　「インターネット上に存在するコンピューターやネットワークを識別するための名前」　

DNS　インターネット上でドメイン名（ドメインネーム※1）を管理・運用するために開発されたシステム

VPN 離れた場所の間を仮想的な専用線でつないで安全なデータ通信を実現する仕組みで、仮想プライベート・ネットワークとも呼ばれる

ブリッジ　ネットワーク中のデータ（の中にあるMACアドレス）を見て、行き先を振り分けてくれる機器のこと

プロキシサーバー　内部のネットワークとインターネットの境界で動作し、両者間のアクセスを代理して行うもの。プロキシとは「代理」という意味がある。　一度閲覧したWebサーバーのデータを一時的に保存しておくことができる。

SMTP メールを相手のメールサーバーまで届ける仕組み

POP　メールを取り込むための手順 メールをサーバーに残さない

IMAP 「POP」と同じ、メールを取り込むための手順 メールをサーバーに残す

セッション　通信の開始から終了までを指す

クッキー　Webサイトにアクセスした際、ユーザー情報をブラウザで一時的に保存し、同じユーザーが再アクセスした時に、Cookieによりサーバー側で同一ユーザーであることを識別する。

キャッシュ　よく使うデータを取り出しやすいところに準備しておく仕組み

チューニング　良い感じになるように、何かを調整すること　通信の状態が良くなるように電波状況を調整したり、たくさんのデータを処理するためにプログラムの作りを見直したり、あるいは、パソコンさんの部品を性能の良い部品と取り換えたりするのも、すべて「チューニング」と呼べる

スロークエリ　データベースにおいて、時間のかかっているSQL　パフォーマンスの悪い SQL を特定するための基本となる

インデックス　SEO的な意味で言えば検索エンジンの中に記録されているホームページの情報
　　　　　　　データベース的な意味で言えば検索を速くするための仕組み

フロー　何かの「流れ」のこと　例　データフロー　＝「データの流れ」

単体テスト　開発したシステムを機能ごとに検証

結合テスト　システムにおける複数の機能を組み合わせて動作を検証

チケット管理ツール　タスク管理ツール　プロジェクト毎に発生する各種タスクを発行して、制作者に対して仕事を依頼するためのもの

タスク　誰か（もしくは何か）がやるべき「お仕事」

Git　ファイルのバージョン管理が簡単にできるツール

SourceTree　Gitを簡単に使えるようにしたツール　GUI形式でGitを使えるツール

SVN バージョン管理システムのひとつでありファイルの変更履歴を管理してくれるシステムのひとつ

コンフリクト 複数のプログラムが同じデータとかファイルを同じタイミングで使おうとして「俺が使うんだ～」「い～や、私が使うのよ～」になっている状態のこと 日本語で「競合」と表現される場合もある

デグレ　あれ？前の方が良かったんじゃね？」な状態のこと　
　　　　プログラムにおいて、直したはずのバグが復活していたり、前はなかったはずのバグがあったり、実　　　　　
装したはずの機能がなくなっている状態のこと

マージツール　複数のファイルやデータ、プログラムなどを、決められたルールに従って一つに統合することをマージという。
